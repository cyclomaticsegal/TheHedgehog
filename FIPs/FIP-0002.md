## Supersedes

This FIP supersedes the Dexter integration approach described in FIP-0001. FIP-0001 proposed embedding Dexter as an external TypeScript/Bun subprocess via `egui_term` (Option 3). That approach was abandoned in favour of a full native Rust port of Dexter's functionality, making it a first-class part of The Hedgehog rather than an external dependency.

FIP-0001 remains authoritative for:
- The app rename to "The Hedgehog"
- The three-tab workspace structure (Regime Shift, Dexter, 51Folds)

FIP-0001 is no longer authoritative for:
- Dexter integration options (Options 1–3)
- The `egui_term` / `notify` / subprocess architecture
- The scratchpad file watcher approach

## Goal

Port the core functionality of [Dexter](https://github.com/virattt/dexter) — an autonomous financial research agent written in TypeScript — into native Rust as an integral module of The Hedgehog. The agent runs on a background thread and streams structured events to the Dexter tab's terminal-style UI.

### Why native port over subprocess

1. **No external runtime dependency** — eliminates the requirement for Bun/Node.js and a separate Dexter installation
2. **Shared infrastructure** — reuses existing LLM calling pipeline (`ai.rs`), HTTP client (`reqwest`), and settings/API key management
3. **Tighter integration** — agent events flow directly via `mpsc::channel`, no filesystem watching or process coordination
4. **Single binary** — the entire application ships as one executable
5. **Extensibility** — the agent can be evolved independently as part of The Hedgehog without tracking upstream Dexter changes

### Agent Loop (`agent.rs`)

ReAct (Reasoning + Acting) loop ported from `agent/agent.ts`:

1. Build system prompt with tool descriptions and research methodology
2. Send user query + message history to LLM with tool definitions
3. Parse response for text content (thinking) and tool calls
4. If no tool calls → final answer, break
5. Execute tool calls via `tools::execute_tool()`
6. Record results in scratchpad, append to message history
7. Microcompact if 8+ tool results or 80K+ estimated tokens
8. Full LLM compaction if context exceeds 60K tokens
9. Loop back to step 2 (max 10 iterations)

Events streamed to UI via `mpsc::channel<DexterEvent>`:
- `Thinking(text)` — agent reasoning
- `ToolStart { name, args }` — tool invocation beginning
- `ToolEnd { name, result, success }` — tool completion
- `ToolLimit { name, warning }` — soft limit warning
- `Microcompact { cleared }` — old results cleared
- `Compaction { phase }` — full context summarization
- `Answer(text)` — final response (markdown)
- `Error(text)` — error message
- `Done` — agent finished

### Finance API (`finance.rs`)

All endpoints target `https://api.financialdatasets.ai` with `x-api-key` header authentication. Responses are formatted into markdown tables via `format.rs`.

### Formatters (`format.rs`)

Ported from Dexter's `formatters.ts`. Converts raw JSON API responses into compact markdown tables optimised for LLM context:
- `fmt_num()` — 1.2T, 3.4B, 5.6M, 7.8K
- `fmt_pct()` — decimal to percentage (0.15 → "15.0%")
- `fmt_price()` — "$X.XX"
- Per-statement-type table formatters (income, balance, cash flow)
- Formatters for prices, key ratios, news, earnings, estimates, insider trades, segments

### Scratchpad (`scratchpad.rs`)

Ported from `agent/scratchpad.ts`. In-memory state tracking (no JSONL file — runs within the app process):
- Tool call counting with soft limits (default: 3 calls per tool)
- Query similarity detection via Jaccard word overlap (threshold: 0.7)
- Warnings emitted as `DexterEvent::ToolLimit` (never blocks)

### Compaction (`compact.rs`)

Ported from `agent/compact.ts` + `agent/microcompact.ts`:

**Microcompaction** (no LLM call):
- Triggers at 8+ compactable tool result messages OR 80K+ estimated tokens
- Clears oldest tool results, keeps most recent 4
- Token estimation: `ceil(chars / 3.5)`

**Full compaction** (LLM call):
- Triggers when total context exceeds 60K tokens
- Calls LLM with compaction prompt to summarize conversation
- Replaces message history with summary + "Continue the research"
- Tracks consecutive failures (stops trying after 3)
- Falls back to message truncation on failure

### LLM Integration

Supports both Anthropic and OpenAI with tool calling:

**Anthropic**:
- Converts tool definitions to Anthropic format (`input_schema` instead of `parameters`)
- Converts messages to Anthropic format (content blocks, `tool_use`/`tool_result`)
- Merges consecutive same-role messages (Anthropic requirement)
- Normalizes response back to common format

**OpenAI**:
- Native function-calling format (tools array in request)
- Response parsed directly from `choices[0].message`

Both providers reuse API keys from the existing settings system (same keys as Regime Shift AI panel).

### UI Integration (`app.rs`)

The Dexter tab provides a terminal-style interface:
- **Bottom input bar**: `>` prompt, text input, Send button, spinner when in-flight
- **Central panel**: scrolling log of `DexterLogEntry` items, colour-coded by type
  - User queries in green
  - Thinking in muted grey
  - Tool calls in primary text colour
  - Tool results in secondary text colour
  - Answers rendered as markdown via `egui_commonmark`
  - Errors in red
- **Polling**: `poll_dexter()` runs every frame, drains events from the background thread

### API Keys Required

| Key | Environment Variable | Purpose |
|-----|---------------------|---------|
| Anthropic or OpenAI | `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` | LLM for agent reasoning (shared with Regime Shift tab) |
| Financial Datasets | `FINANCIAL_DATASETS_API_KEY` | All financial data tools |
| Exa | `EXASEARCH_API_KEY` | Web search |

### What Was Not Ported

| Dexter Feature | Reason |
|---------------|--------|
| WhatsApp gateway | Not relevant to desktop app |
| Browser automation (Playwright) | Heavy dependency, no Rust equivalent |
| Filesystem tools (read/write/edit) | Not relevant to financial research scope |
| Memory system (SQLite + vectors, embeddings, MMR, temporal decay) | Can be added later; agent works without persistent memory |
| Skills system (SKILL.md extensibility) | Can be added later |
| Cron / heartbeat tools | Operational tooling, not needed for interactive use |
| X/Twitter search | Can be added later with API key |
| Streaming LLM responses | Uses blocking calls; streaming can be added for progressive UI updates |
| LangSmith evaluation framework | Development tooling |
| Ink/React TUI components | Replaced by egui terminal-style UI |
| Multiple LLM providers beyond Anthropic/OpenAI | Gemini, xAI, OpenRouter, Ollama, etc. can be added later |

## Dependencies Changed

**Removed** (from FIP-0001):
- `egui_term` — no external terminal embedding
- `notify` — no filesystem watching

**No new dependencies added.** The port uses only existing crates (`reqwest`, `serde_json`, `chrono`, `anyhow`).
